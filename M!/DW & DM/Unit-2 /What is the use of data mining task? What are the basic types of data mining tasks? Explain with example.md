# What is the Use of Data Mining Task? What are the Basic Types of Data Mining Tasks? Explain with Example. (15 Marks)

## 1. Use of Data Mining Tasks

A **data mining task** specifies the type of analysis that a user wants to perform on the data.
It defines **what kind of knowledge or patterns should be discovered from the database**. 

Data mining tasks help in:

* Discovering **hidden patterns and relationships** in large datasets
* **Analyzing trends and behaviors** in data
* Supporting **decision making and prediction**
* Extracting **useful knowledge** from databases, data warehouses, or other repositories. 

Data mining functionalities are used to specify the **type of patterns to be discovered** during the mining process. 

Generally, data mining tasks are classified into **two major categories**:

1. **Descriptive data mining tasks**
2. **Predictive data mining tasks** 

---

# 2. Basic Types of Data Mining Tasks

## 1. Descriptive Data Mining Tasks

Descriptive tasks describe the **general properties or characteristics of the data** stored in the database. 

They help to understand patterns and relationships within the data.

Examples include:

* Characterization
* Discrimination
* Association analysis
* Clustering
* Outlier analysis
* Evolution analysis

---

### a) Data Characterization

Data characterization summarizes the **general characteristics of a target class of data**. 

Example:
Analyzing the characteristics of customers whose sales increased by 10% last year.

The result may be presented using:

* Charts
* Tables
* Data cubes
* Characteristic rules.

---

### b) Data Discrimination

Data discrimination compares the **features of one class with another class**. 

Example:
Comparing the characteristics of customers who buy expensive products with those who buy cheaper products.

---

### c) Association and Correlation Analysis

Association analysis finds **relationships between items that frequently occur together** in a dataset. 

Example:
In a supermarket database:

```
buys(X, “computer”) ⇒ buys(X, “software”)
```

This means customers who buy computers are also likely to buy software.

---

### d) Clustering

Clustering groups data objects into clusters such that **objects within the same cluster are similar** and objects in different clusters are dissimilar. 

Example:
Grouping customers based on purchasing behavior.

### e) Outlier Analysis

Outlier analysis identifies **data objects that significantly differ from the rest of the data**. 

Example:
Detecting fraudulent credit card transactions.

---

### f) Evolution Analysis

Evolution analysis studies **patterns and trends over time**. 

Example:
Analyzing stock market trends over several years.

---

# 3. Predictive Data Mining Tasks

Predictive tasks are used to **predict future values or unknown data** based on existing data. 

Examples include:

* Classification
* Prediction

---

### a) Classification

Classification is the process of **finding a model that can predict the class label of data objects**. 

Example:
Classifying emails as **spam or non-spam**.

### b) Prediction

Prediction estimates **continuous numerical values** based on previous data. 

Example:
Predicting future **stock prices or sales revenue**.

---

# Conclusion

Data mining tasks define the **type of knowledge to be extracted from data**.
They are broadly classified into **descriptive tasks**, which describe data patterns, and **predictive tasks**, which forecast future trends. These tasks help organizations discover useful insights and support effective decision making.
